# GitHub Releases Auto-Updates (Tauri v2)

This app is set up to use the Tauri updater plugin with GitHub Releases.

## 1) Configure updater endpoint + public key

- Endpoint is already set to:
  - `https://github.com/vo1x/GDExplorer/releases/latest/download/latest.json`
- You must set the updater public key in `src-tauri/tauri.conf.json`:
  - `plugins.updater.pubkey`

## 2) Generate signing keys (one-time)

Run:

`npx tauri signer generate`

This prints:

- a **private key** (keep secret)
- a **public key** (put into `src-tauri/tauri.conf.json` `plugins.updater.pubkey`)

## 3) Add GitHub repo secrets

Add these repository secrets (Settings → Secrets and variables → Actions):

- `TAURI_PRIVATE_KEY` = the private key generated by `tauri signer generate`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` = the password you set (or empty if you used none)

## 4) Create a release

Tag a version and push it:

- `git tag v0.1.1`
- `git push origin v0.1.1`

The workflow `.github/workflows/release.yml` builds and uploads:

- installers (Windows + macOS)
- updater artifacts (`latest.json` + signatures)

## 5) In-app update check

- The app already listens for the menu event `Check for Updates…` and calls the updater plugin.
- There is also an automatic update check shortly after startup (see `src/App.tsx`).
